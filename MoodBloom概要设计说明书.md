## 1. 引言

### 1.1 编写目的

本概要设计说明书旨在全面描述MoodBloom情绪管理系统的架构设计、模块划分、接口设计和数据存储方案，为后续详细设计和开发工作提供指导依据。本文档遵循GB/T 8567-2006《计算机软件文档编制规范》的要求编制。

### 1.2 背景

MoodBloom是一款面向移动端的情绪记录与管理应用，帮助用户记录每日情绪状态，分析情绪变化趋势，并提供个性化的情绪洞察和建议。系统基于现代化的前端技术栈构建，采用本地数据持久化方案，确保用户隐私安全。

### 1.3 术语定义

| 术语 | 定义 |
|------|------|
| 情绪记录 | 用户在特定时间点记录的情绪状态及相关信息 |
| 情绪分值 | 用于量化情绪状态的数值，范围1-5 |
| 情绪洞察 | 基于用户数据统计分析得出的规律性结论 |
| localStorage | 浏览器本地存储API，用于数据持久化 |

### 1.4 情绪体系定义

本系统采用5种情绪分类体系：

| 情绪Key | 名称 | Emoji | 主题色 | 背景色 | 分值 |
|---------|------|-------|--------|--------|------|
| elated | 开心 | 😄 | #22c55e | #dcfce7 | 5 |
| good | 良好 | 😊 | #84cc16 | #f0fdf4 | 4 |
| neutral | 平静 | 😐 | #f59e0b | #fffbeb | 3 |
| low | 低落 | 😔 | #3b82f6 | #eff6ff | 2 |
| rough | 糟糕 | 😣 | #ef4444 | #fef2f2 | 1 |

## 3. 接口设计

### 3.1 数据存储接口

#### 3.1.1 localStorage存储结构

**Key**: `moodRecords`

**数据类型**: JSON数组

**单条记录结构**:

```javascript
{
  "date": "2026/6/25",           // 日期字符串 (toLocaleDateString格式)
  "mood": {                      // 情绪对象
    "key": "elated",             // 情绪唯一标识
    "name": "开心",               // 情绪名称
    "emoji": "😄",               // 情绪图标
    "color": "#22c55e",          // 主题色
    "bgColor": "#dcfce7",        // 背景色
    "textColor": "#166534"       // 文字色
  },
  "strength": 5,                 // 情绪分值 (1-5)
  "note": "今天很开心！",         // 用户备注
  "time": 1750876800000,         // Unix时间戳
  "timeOfDay": "morning",        // 时段标签 (morning/afternoon/evening)
  "hasExercise": true,           // 是否运动
  "hasSocial": false             // 是否有社交
}
```

| 路由路径 | 组件 | 描述 |
|----------|------|------|
| / | Record | 情绪记录页面（默认首页） |
| /calendar | Calendar | 日历查看页面 |
| /analysis | Analysis | 数据分析页面 |
| /insight | Insight | 情绪洞察页面 |

### 3.3 模块间接口

#### 3.3.1 情绪数据访问接口

```javascript
// 读取所有记录
const getRecords = () => {
  return JSON.parse(localStorage.getItem('moodRecords') || '[]')
}

// 保存记录
const saveRecord = (record) => {
  const oldRecords = getRecords()
  oldRecords.push(record)
  localStorage.setItem('moodRecords', JSON.stringify(oldRecords))
}
```

## 6. 功能模块详细设计

### 6.1 情绪记录模块 (Record)

#### 6.1.1 功能描述

用户选择当前情绪状态，添加可选备注和标签，然后保存到本地存储。

#### 6.1.2 处理流程

```
开始
  │
  ├─> 选择情绪 ──┐
  │              │
  │  ┌─< ────────┘ (未选择)
  │  │
  │  ↓ (已选择)
  ├─> 填写备注 (可选)
  │
  ├─> 选择时段标签 (自动/手动)
  │    - 6:00-12:00 → 早晨
  │    - 12:00-18:00 → 下午
  │    - 18:00-24:00 → 晚上
  │
  ├─> 选择运动标签 (可选)
  │
  ├─> 选择社交标签 (可选)
  │
  ├─> 点击保存按钮
  │
  ├─> 构造记录对象
  │    - date: 当前日期
  │    - mood: 情绪对象
  │    - strength: 情绪分值
  │    - note: 备注文本
  │    - time: 时间戳
  │    - timeOfDay: 时段
  │    - hasExercise: 运动标记
  │    - hasSocial: 社交标记
  │
  ├─> 读取localStorage
  │
  ├─> 追加新记录到数组
  │
  ├─> 保存回localStorage
  │
  ├─> 显示保存成功状态
  │
  └─> 重置表单 (1.5秒后)
结束
```

#### 6.1.3 时段判断逻辑

```javascript
const getTimeOfDay = () => {
  const hour = new Date().getHours()
  if (hour >= 6 && hour < 12) return 'morning'  // 早晨
  if (hour >= 12 && hour < 18) return 'afternoon'  // 下午
  return 'evening'  // 晚上
}
```

### 6.3 分析模块 (Analysis)

#### 6.3.1 功能描述

统计用户的情绪数据，以图表形式展示趋势和分布。

#### 6.3.2 统计指标

| 指标 | 计算方法 |
|------|----------|
| 平均情绪 | 所有记录strength之和 / 记录数量 |
| 连续打卡 | 从最新记录向前追溯连续有记录的天数 |
| 总记录天数 | 去重后的日期数量 |

#### 6.3.3 图表配置

**趋势面积图**:
- 类型: 折线图填充
- X轴: 日期
- Y轴: 情绪分值
- 数据源: 时间范围内所有记录

**情绪分布柱状图**:
- 类型: 柱状图
- X轴: 5种情绪类型
- Y轴: 出现次数
- 颜色: 对应情绪主题色

### 6.4 洞察模块 (Insight)

#### 6.4.1 功能描述

基于用户历史数据，通过统计分析生成情绪洞察和建议。

#### 6.4.2 洞察卡片定义

| 卡片 | 类型 | 数据需求 | 分析方法 |
|------|------|----------|----------|
| 早晨能量 | 模式 | timeOfDay字段 | 按时段分组计算平均分 |
| 运动有益 | 习惯 | hasExercise字段 | 运动日vs非运动日对比 |
| 周末低落 | 趋势 | 日期字段 | 工作日vs周末对比 |
| 社交提升 | 模式 | hasSocial字段 | 社交日vs独处日对比 |
| 周中低谷 | 趋势 | 日期字段 | 按星期几分组计算 |

#### 6.4.3 洞察算法

**时段分析**:
```javascript
const analyzeTimeOfDay = (records) => {
  const groups = { morning: [], afternoon: [], evening: [] }
  records.forEach(r => {
    if (r.timeOfDay) groups[r.timeOfDay].push(getMoodValue(r))
  })
  return {
    morning: avg(groups.morning),
    afternoon: avg(groups.afternoon),
    evening: avg(groups.evening)
  }
}
```

**运动分析**:
```javascript
const analyzeExercise = (records) => {
  const exercise = records.filter(r => r.hasExercise)
  const rest = records.filter(r => !r.hasExercise)
  return {
    exerciseAvg: avg(exercise),
    restAvg: avg(rest),
    diff: avg(exercise) - avg(rest)
  }
}
```
