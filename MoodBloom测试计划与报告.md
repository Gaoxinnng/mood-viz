## 1. 文档说明

### 1.1 编写目的

本文档依据GB/T 9386-2008《计算机软件测试文档编制规范》编制，包含测试计划和测试报告两部分。旨在系统地规划MoodBloom情绪管理系统的测试工作，并记录测试执行结果、缺陷统计与分析、测试结论。

### 1.2 适用范围

本测试计划适用于MoodBloom情绪管理系统V1.0版本的全部功能模块，包括：
- 情绪记录模块（Record）
- 日历模块（Calendar）
- 数据分析模块（Analysis）
- 情绪洞察模块（Insight）

### 1.3 术语定义

| 术语 | 定义 |
|------|------|
| 测试用例 | 为特定测试目标而设计的一组输入、执行条件和预期结果 |
| 缺陷（Bug） | 软件未能实现规定功能或实现方式与需求不符的问题 |
| 严重级别 | 缺陷对系统影响程度的分级（致命/严重/一般/轻微） |
| 测试覆盖 | 测试用例对系统功能、路径、数据的覆盖程度 |

**BUG-001：跨页面数据不互通**

- **现象**：记录页保存的数据，在日历页和分析页不显示
- **根因**：
  1. localStorage的key不一致（Record用'moodRecords'，Calendar用'mood_data'）
  2. 数据格式不一致（Record保存mood对象，其他页面期望不同结构）
  3. 日期格式不一致（Record用'toLocaleDateString()'，Calendar用自定义格式）
  4. 情绪key映射不完整
- **修复方案**：
  1. 统一localStorage key为'moodRecords'
  2. 统一数据格式，以Record页为准
  3. 统一日期格式为'toLocaleDateString()'
  4. 补全情绪key映射
- **影响范围**：Record、Calendar、Analysis三个页面

**BUG-002：emoji表情不显示**

- **现象**：情绪选择按钮和日历日期上的emoji不显示
- **根因**：
  1. 数据结构中emoji字段名不一致（保存用'mood.emoji'，读取用'mood.icon'）
  2. 情绪映射表中缺少emoji字段
- **修复方案**：
  1. 统一emoji字段名为'emoji'
  2. 补全情绪映射表中的emoji字段
  3. 确保保存完整的mood对象包含emoji
- **影响范围**：Record、Calendar页面

**BUG-003：洞察数据不实时更新**

- **现象**：保存新记录后，洞察页面数据不更新
- **根因**：useEffect依赖数组缺少location，导致路由变化时不重新读取数据
- **修复方案**：在useEffect依赖数组中添加location.pathname
- **影响范围**：Analysis、Insight页面

**BUG-004：折线图不显示**

- **现象**：分析页面的趋势折线图没有显示
- **根因**：数据key不匹配（使用中文标签访问英文key）
- **修复方案**：重构lineData函数，使用正确的key并过滤空数据集
- **影响范围**：Analysis页面

**BUG-005：情绪名称统计为空**

- **现象**："最常出现的情绪"名称为空，只显示次数
- **根因**：统计代码中取情绪名称的字段名错误（用了label而非name）
- **修复方案**：修正字段名为name
- **影响范围**：Analysis页面

**BUG-006：Tailwind opacity语法错误**

- **现象**：样式不生效，控制台报错"Cannot apply unknown utility class"
- **根因**：Tailwind CSS v4不支持`bg-white/80`语法
- **修复方案**：替换为`bg-[rgba(255,255,255,0.8)]`格式
- **影响范围**：全局样式

**BUG-007：洞察文案固定不变**

- **现象**：分析页"情绪小洞察"文案固定，不根据数据变化
- **根因**：文案硬编码，没有根据统计结果动态生成
- **修复方案**：根据平均情绪和最常出现情绪动态生成文案
- **影响范围**：Analysis页面

**BUG-008：页面加载时有短暂闪烁**

- **现象**：页面切换时，主题色变化有短暂闪烁
- **根因**：主题色切换时机在组件挂载后，导致初始渲染使用默认样式
- **修复方案**：使用CSS变量和即时样式切换，减少闪烁
- **影响范围**：全局布局
