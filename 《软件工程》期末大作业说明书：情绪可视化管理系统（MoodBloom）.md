# 第一章 概述

## 1.1 编写目的

本说明书是《软件工程》课程期末大作业完整总结文档，完整记录情绪可视化管理系统（MoodBloom）从立项可行性分析、需求梳理、架构设计、详细编码到系统测试全流程软件工程实践工作。文档整合项目全部产出，梳理各阶段核心工作、成果、问题与总结，供课程指导教师、课程评审教师审阅打分，同时完整留存个人前端软件工程项目实践档案。

## 1.2 项目背景

1. **项目名称**：情绪可视化管理系统（MoodBloom）

2. **项目来源**：《软件工程》课程期末自主选题大作业

3. **项目意义与应用场景**
当下大学生、年轻职场人群心理压力普遍偏大，情绪波动频繁，但传统纸质手账、普通备忘录记录情绪存在操作繁琐、无自动数据分析、无法直观查看情绪规律等痛点。本系统为纯前端 Web 轻量化工具，无需注册登录、所有数据本地存储，用户可快速记录每日情绪，系统自动生成情绪趋势图表、多维度情绪洞察，帮助用户自我觉察、提升情绪管理能力，适用于学生日常心理记录、心理咨询辅助记录等场景。

4. **开发团队**：单人独立开发（高颖欣，数媒 2401）

# 第二章 项目概述

## 2.1 选题背景与意义

### 选题背景

当代青年群体学业、工作压力持续增加，心理健康需求持续上涨，市面上多数情绪记录工具存在需要注册、数据云端上传、操作复杂、存在隐私泄露风险等问题。而纸质记录、Excel 表格手动统计方式效率极低，无法自动生成可视化图表与情绪规律分析。结合课程软件工程实践要求，选取纯前端情绪可视化工具作为开发项目，完整落地软件全生命周期流程。

### 应用价值

1. **用户价值**：零门槛使用，浏览器直接打开，数据仅保存在本地浏览器，保护隐私；一键记录情绪，自动完成统计、图表、智能洞察，降低情绪记录坚持成本。

2. **学习价值**：完整实践软件工程标准流程，掌握需求分析、分层设计、系统测试全套规范，熟练 React 前端技术栈、本地数据持久化、数据统计算法开发。

3. **社会价值**：轻量化免费心理健康辅助工具，无医疗资质风险，普及情绪自我觉察理念，适合高校心理健康宣传使用。

## 2.2 项目目标

1. **功能目标**：完成四大核心模块（情绪记录、情绪日历、数据分析、情绪洞察），实现情绪录入、月历展示、多维度图表、五大类情绪规律自动分析功能，所有 P0 级核心需求全部落地。

2. **技术目标**：基于 React+Vite 搭建纯前端 SPA，使用 localStorage 完成本地数据持久化，兼容 PC、Android、iOS 多终端主流浏览器，性能指标达标。

3. **工程目标**：严格遵循国标软件文档规范，产出全套标准化软件工程文档，完整完成测试流程，所有缺陷 100% 修复。

4. **学习目标**：熟练掌握软件生命周期各阶段工作方法，掌握前端数据统计、图表渲染、异常容错处理开发思路。

## 2.3 技术选型

|技术领域|选型方案|选型理由|
|---|---|---|
|前端框架|React 18.x|组件化开发模式成熟，生态完善，适配 SPA 单页应用开发，学习资源丰富|
|构建工具|Vite 5.0+|对比 Webpack 热更新速度大幅提升，打包体积更小，适合学生快速迭代开发|
|路由工具|react-router-dom v6|官方标准路由库，实现页面无刷新切换，适配底部导航多页面场景|
|样式框架|Tailwind CSS 4.x|原子化 CSS，无需自定义大量样式文件，快速实现响应式移动端适配；新版本对 CSS 变量支持更完善|
|图表库|Chart.js + react-chartjs-2|轻量开源可视化库，支持折线、柱状图，适配情绪数据统计展示|
|本地存储|浏览器原生 localStorage|无需后端服务，实现数据本地持久化，完全保护用户隐私|
|部署平台|Netlify|静态网站免费托管，一键自动部署，无需服务器运维|

## 4.2 遇到的问题与解决方案

|问题描述|根因分析|解决方案|解决效果|
|---|---|---|---|
|页面间 localStorage 数据不同步，日历/分析页无法读取新记录|1. 存储 key 不一致（Record 用'moodRecords'，Calendar 用'mood_data'）<br/>2. 数据格式不一致（Record 保存完整 mood 对象，其他页面期望不同结构）<br/>3. 日期格式不一致（Record 用'toLocaleDateString()'，Calendar 用自定义格式）<br/>4. 情绪 key 映射不完整|统一全局存储 key 为'moodRecords'，标准化单条记录数据结构，统一日期格式化规则，路由切换时重新读取存储数据|彻底解决跨页面数据不一致问题，所有页面实时同步最新记录|
|Tailwind 旧版 opacity 语法在新版报错，全局样式失效|Tailwind CSS v4 不再支持`bg-white/80`缩写语法|替换为`bg-[rgba(255,255,255,0.8)]`固定透明写法，重构全局卡片样式工具类|页面样式全部正常，无控制台报错|
|情绪 emoji 图标部分浏览器无法正常渲染|1. 数据结构中 emoji 字段名不一致（保存用'mood.emoji'，读取用'mood.icon'）<br/>2. 情绪映射表中缺少 emoji 字段|统一 emoji 字段名为'emoji'，确保保存完整的 mood 对象包含 emoji，补全情绪映射表中的 emoji 字段|日历、记录页表情全部正常展示|
|连续打卡、周中情绪均值算法计算逻辑出错|分组计算函数中日期差值计算有误，缺少空数组、边界数据判断容错|重构分组、日期差值计算函数，增加空数组、边界数据判断容错，优先取 mood.value 再取 strength|统计数据计算 100% 准确，无 NaN、数值错乱问题|
|洞察数据不实时更新|useEffect 依赖数组缺少 location，导致路由变化时不重新读取数据|在 useEffect 依赖数组中添加 location.pathname，路由切换时触发数据重新加载|保存新记录后切换页面，洞察数据即时更新|
|大量记录时图表渲染卡顿|图表渲染逻辑未做数据分页，千条记录一次性渲染|增加数据分页过滤，简化图表渲染逻辑，减少不必要重渲染|千条记录场景图表流畅加载，无页面卡死|
